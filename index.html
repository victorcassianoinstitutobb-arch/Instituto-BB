# Instituto Bruna Braga — Guia de Implantação (Supabase + GitHub + Vercel) Este guia leva o sistema do arquivo local para a nuvem, com dados salvos e login da equipe. Siga na ordem. Não precisa saber programar — é copiar, colar e clicar. --- ## VISÃO GERAL - **Supabase** = banco de dados na nuvem + login (guarda tudo). - **GitHub** = guarda o código com versões. - **Vercel** = publica o site num endereço acessível de qualquer lugar. O sistema funciona em **dois modos**: - **Modo local** (sem configurar nada): abre direto com dados de exemplo. Bom para testar. - **Modo nuvem** (após configurar o Supabase): pede login e salva tudo de verdade. --- ## PARTE 1 — SUPABASE (banco de dados) ### 1.1 Criar as tabelas 1. Acesse seu projeto no [supabase.com](https://supabase.com). 2. No menu lateral, abra **SQL Editor**. 3. Clique em **New query**, cole TODO o conteúdo do arquivo `supabase_schema.sql` e clique em **Run**. 4. Deve aparecer "Success". As tabelas foram criadas com segurança (cada usuário só vê os próprios dados). ### 1.2 Criar o usuário de acesso (login da equipe) 1. No menu lateral, abra **Authentication > Users**. 2. Clique em **Add user > Create new user**. 3. Preencha e-mail e senha (ex: `bruna@institutobrunabraga.com`). Marque **Auto Confirm User**. 4. Repita para cada pessoa da equipe que vai acessar. ### 1.3 Pegar as chaves de conexão 1. No menu lateral, abra **Project Settings > API**. 2. Copie dois valores: - **Project URL** (algo como `https://xxxxx.supabase.co`) - **anon public** key (uma chave longa) ### 1.4 Colar as chaves no sistema 1.
